一、技术方案概览
────────────────────────────────────────────────────────────────────────────────

  方案: React Context 轻量 i18n（不上框架，不额外依赖）

  核心文件:
    src/i18n.js                          # 翻译表 + Context Provider + useLocale hook
    src/App.jsx                          # 加语言切换按钮 + Provider 包裹
    src/components/*.jsx                 # 各组件调用 t(key) 替换硬编码中文

  数据流:
    useLocale() → { locale, t, toggleLocale }
         ↑
    LocaleProvider (在 App.jsx 根层)
         ↑
    initialState: URL ?lang= > localStorage > navigator.language


二、按钮设计
────────────────────────────────────────────────────────────────────────────────

  位置: header 左上角（与右上角 "?" 按钮对称）

  样式:
    - 方形（非圆形）
    - bg-[#ff00ff] border-2 border-black text-black  font-black
    - hover:bg-black hover:text-[#ff00ff]
    - 尺寸与 "?" 按钮一致: w-7 h-7 sm:w-9 sm:h-9
    - 字体: 12px (移动端) / 14px (桌面端)
    - 显示当前语言: "中" (当前en时点击变zh) / "En" (当前zh时点击变en)
    - position: absolute, top-2 left-2 sm:top-3 sm:left-3

  行为:
    - 点击调用 toggleLocale()
    - 切换后写入 localStorage (key: kims_video_lang)
    - 切换后通过 history.replaceState 更新 URL 参数 ?lang=zh|en

  CSS 优先级: 因按钮在 header 内且 header 有 relative z-10，
  按钮用 absolute 定位在 header 的左上角。

  5.1 src/i18n.js — 新增
  ────────────────────────────────────────────────────────────────────────────
    - 定义 messages 对象（zh / en 两个语言包）
    - 创建 LocaleContext
    - LocaleProvider 组件:
        state 初始化逻辑:
          1. URL 读取: new URLSearchParams(window.location.search).get('lang')
          2. localStorage 读取: localStorage.getItem('kims_video_lang')
          3. 浏览器语言: navigator.language?.startsWith('zh') ? 'zh' : 'en'
        setLocale 副作用:
          - 写入 localStorage
          - 通过 history.replaceState 更新 URL 参数
    - useLocale() hook: useContext(LocaleContext)
    - 导出 { LocaleProvider, useLocale }

  5.2 src/App.jsx — 修改
  ────────────────────────────────────────────────────────────────────────────
    A. import:
         import { LocaleProvider, useLocale } from "./i18n";

       或之内，确保所有子组件可访问 Context)
         // 方式: LocaleProvider 放在最外层，内部是现有 App 内容

二、核心改动
────────────────────────────────────────────────────────────────────────────────

  2.1 prompts.js — 所有 builder 函数新增 locale 参数，双语模板

    每个 builder 结构:
      export const buildXxxPrompt = (..., locale = 'zh') => {
        if (locale === 'en') return `...English prompt...`;
        return `...Chinese prompt...`;
      };

    受影响函数:
      - buildSearchPrompt(query, locale)
      - buildQuestionPrompt(primaryMovie, secondaryMovie, locale)
      - buildRecommendationPrompt(primaryMovie, secondaryMovie, answersText, locale)
      - buildFillPrompt(excludeStr, position, locale)
      - buildReplacePrompt(primaryMovie, secondaryMovie, answersText, excludeList, isNiche, isControversial, locale)
      - buildRepairPrompt(rec, locale)

    Schema 语言切换:
      Schema 中的 description 字段在 en 时切换为英文描述。
      由于 schema 是常量对象，每个 schema 导出为函数返回动态对象:
        export const searchSchema = (locale) => ({ ...locale === 'en' ? English descriptions : Chinese descriptions })

  2.2 推荐理由字数放宽至 50 字

    Schema description:
      zh: "详细的推荐理由（50字左右）"
      en: "Detailed recommendation reason (around 50 words)"

    English prompt 追加:
      "Each reason should be approximately 50 words."

  2.3 api.js — TMDB 调用支持语言参数

    fetchMovieByTmdbId(tmdbId, locale):
      → 请求 body 增加 language: locale === 'en' ? 'en-US' : 'zh-CN'

    verifyMovieTmdbId(title, year, locale):
      → 同上

    repairRecommendationFields(rec, generateAIContent, locale):
      → 修复提示词 locale-aware

  2.4 useMovieEngine.js — 接入 useLocale()

    顶部 import { useLocale } from "../i18n";
    函数内 const { t, locale } = useLocale();

    具体替换:
      - 加载文案: setLoadingMessage(t('loading.xxx'))
      - 提示词调用: buildQuestionPrompt(primary, secondary, locale)
      - TMDB 调用: fetchMovieByTmdbId(id, locale)
      - 分享文本: t('share.text', { title: ... })
      - alert 弹窗: t('alert.copied')

  2.5 i18n.jsx — 新增翻译 key

    新增 key 清单:
